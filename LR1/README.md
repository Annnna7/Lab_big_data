# Лабораторная работа №1: Introduction to Apache Spark

**Цель работы:** Изучение базовых операций Apache Spark для обработки больших данных на примере анализа данных о поездках велосипедов в Сан-Франциско (2013-2015 гг.).

## Этапы выполнения работы

### 1. Настройка окружения
В начале работы была создана SparkSession - точка входа для всех операций Spark:
- Установлено автоматическое определение схемы данных (inferSchema)
- Настроена адаптивная оптимизация запросов
- Данные загружены в распределенные DataFrame

### 2. Предобработка данных
Перед анализом выполнено приведение типов данных:
- `duration` → целочисленный тип (int)
- `lat`, `long` → вещественный тип (double)

## Результаты анализа

### Задача 1: Найти велосипед с максимальным временем пробега

**Метод решения:**
Применена группировка по идентификатору велосипеда (`bike_id`) с суммированием длительности всех поездок для каждого велосипеда. Результат отсортирован по убыванию суммарного времени.

**Полученный результат:**
- **ID велосипеда-рекордсмена**: 535
- **Общее время пробега**: 18 611 693 секунды

---

### Задача 2: Найти наибольшее геодезическое расстояние между станциями

**Метод решения:**
Для расчета расстояния между двумя точками на земной поверхности использована **формула гаверсинуса**:
distance = 2R·arcsin(√(sin²(Δφ/2) + cos φ₁·cos φ₂·sin²(Δλ/2)))

где R = 6371 км (средний радиус Земли), φ - широта, λ - долгота.

Техническая реализация:
1. Выполнен crossJoin таблицы станций для создания всех возможных пар
2. Добавлен фильтр `st1.id < st2.id` для исключения дублирования и самосоединений
3. Координаты переведены из градусов в радианы
4. По формуле рассчитано расстояние для каждой пары
5. Выбрана пара с максимальным значением

**Полученный результат:**
- **Максимальное расстояние**: 69.92 км
- **Станция A**: SJSU - San Salvador at 4th (Сан-Хосе)
- **Станция B**: Embarcadero at Sansome (Сан-Франциско)

### Задача 4: Найти количество велосипедов в системе

**Метод решения:**
Применен метод `distinct().count()` к колонке `bike_id`, который возвращает количество уникальных значений в распределенной коллекции.

**Полученный результат:**
- **Общее количество велосипедов**: 700 единиц

---

### Задача 5: Найти пользователей потративших на поездки более 3 часов

**Метод решения:**
Поскольку в данных отсутствует прямая идентификация пользователей, в качестве прокси-переменной использован почтовый индекс (`zip_code`). Выполнено:
1. Фильтрация записей с непустым zip_code
2. Группировка по zip_code с суммированием времени поездок
3. Фильтрация записей с суммарным временем > 10 800 секунд (3 часа)
4. Сортировка по убыванию суммарного времени
